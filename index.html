#2: Aren't numbers just numbers?

JavaScript numbers are all floating point, stored according to the IEEE 754 standard.

Numbers are stored in a binary format, in 64 bits.

.1 + .2 = ?

.1 + .2 = 0.30000000000000004
Floating Point precision may be able to represent a number that has large magnitude but not hold precision to a number of small magnitude

#4: Why are there so many equals signs?

' == ' vs ' === '

==

When doing comparisons, the double-equals will return true if the two comparison items are equal. But if the items are two different types, JavaScript will coerce the types to be equals.

===

Whenever you use triple-equals, you’re doing an exact comparison of the actual values. You’re ensuring the values are ‘strictly equal’ or ‘identically equal’.
